Getting Started

The purpose for this document is to explain the use of the API, developed in this assignement. It is an RESTful API, that follows the CRUD principles, that allows the user to Create, Retrieve, Update and Delete content from our database, which stores items sold at Einstein Heijn, and their properties, like; product name, origin, best before date, and an image. This is achieved by using the POST, GET, PUT, and DELETE, http methods, respectively. Their functionality is explained below:

GET Entirety of Stocklist

Endpoint: http://localhost:8080/getAll

This will return the entire inventory in the stock list.

HTTP method: GET

Response format: JSON

Request Parameters

Since this method will get all the inventory, no parameter need to be passed that would

GET Specific Item

Endpoint: GET http://localhost:8080/getSpecific/{id}

This will return the item speficied by the given id in the URL, and show its properties: product name, origin, best before date, and its image.

Request Parameters

Header: none

Path: none

Query: none

Response format

On success, data will be returned in the JSON format. Success code, 200 in the response header.
