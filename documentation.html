Getting Started

The purpose for this document is to explain the use of the API, developed in this assignement. It is an RESTful API, that follows the CRUD principles, that allows the user to Create, Retrieve, Update and Delete content from our database, which stores items sold at Einstein Heijn, and their properties, like; product name, origin, best before date, and an image. This is achieved by using the POST, GET, PUT, and DELETE, http methods, respectively. Their functionality is explained below:

Create Item in Stocklist

Endpoint: http://localhost:8080/create

This will create an item with the, based on the given parameter input:

HTTP method: POST

Response format: JSON

Parameters:

origin
- Where does the product come from
best_before_date
- When is the product going to be bad?
product
- Name of the produce
amount
- Input the quantity of the product (kg or any other unit)
image
- Input URI of the image to be displayed.

Example: POST http://localhost:8080/create?origin=NL&best_before_date=11.03.2022&product=marshmallows&amount=500kg&image=https://hips.hearstapps.com/hmg-prod.s3.amazonaws.com/images/full-frame-of-white-mini-marshmallows-royalty-free-image-899983790-1542738340.jpg?crop=1xw:0.75xh;center,top&resize=1200:*

Retrieve Entirety of Stocklist

Endpoint: http://localhost:8080/getAll

This will return the entire inventory in the stock list.

HTTP method: GET

Response format: JSON

Parameters: none.

Since this method will get all the inventory, no parameter need to be passed that would identify item.

Example: GET http://localhost:8080/getAll

GET Specific Item

Endpoint: GET http://localhost:8080/getSpecific/{id}

This will return the item speficied by the given id in the URL, and show its properties: product name, origin, best before date, and its image.

Request Parameters

Header: none

Path: none

Query: none

Response format

On success, data will be returned in the JSON format. Success code, 200 in the response header.
